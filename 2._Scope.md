# Scope

The Open Component Model (OCM) specification is an open standard for describing software-bill-of-deliveries (SBOD). It provides a technology-agnostic, machine-readable format with a globally unique identity scheme for software artifacts throughout the entire software lifecycle management process.

The specification covers:

* A component model for describing versioned sets of software artifacts, including components, component versions, resources, sources, and references
* Processing operations for referencing, signing, verification, and normalization of component versions
* An abstract persistence layer with storage backend mappings
* An extension framework for artifact types, access method types, storage backends, and algorithms

The specification does not cover the building or deployment of software artifacts.

Any changes of Scope are not retroactive.
